# 个人副脑产品介绍文案

## 一句话介绍

个人副脑是一个本地优先的智能协作控制台，帮助个人开发者长期记住项目背景、理解本地文件和代码、规划任务、调用工具，并在关键动作前请求确认。

当前迭代正在把它推进成一个 Jarvis 风格的“可对话中枢”：用户输入任务后，它会先理解目标、检索记忆和上下文、选择能力模块、生成计划，并明确哪些动作需要确认。

## 它适合谁

适合需要长期处理代码、资料、浏览器调试、开源项目研究和个人知识管理的个人用户。它不是单纯聊天窗口，也不是一堆 AI 工具安装包，而是把记忆、上下文、工具、权限和运行日志组织在一起的本地工作脑。

## 它能做什么

- 记录长期偏好、项目事实、失败经验和可复用结论。
- 搜索本地文件、代码和文档，帮助任务快速建立上下文。
- 研究 GitHub 开源项目，先镜像、审计、固定版本，再决定是否接入。
- 在确认后调用浏览器、DevTools、终端、工作区等工具。
- 把每次任务的计划、执行结果和学习沉淀成可追踪记录。
- 提供指数基金教育模块，用于风险画像、配置模板和再平衡学习。
- 在网页控制台中输入任务，生成结构化的 Jarvis 计划。

## 它不会擅自做什么

- 不会未经确认写文件、删除文件或执行高风险命令。
- 不会自动发送邮件、转发邮件或读取隐私数据。
- 不会把密钥、授权码或本地路径暴露给访客模式。
- 不会直接修改第三方开源项目源码作为默认集成方式。
- 不会把教育性投资分析包装成个人化投资建议。

## 当前能力状态

- 长期记忆：已启用。
- 本地上下文搜索：已启用。
- 运行日志：已启用。
- GitHub 项目注册表：已启用。
- Jarvis 对话中枢：实验中，支持文本任务规划和能力路由。
- Chrome DevTools MCP：实验中，需要用户确认。
- claude-context / Agentmemory / Mirage：作为镜像或适配器候选持续验证。
- Gmail / 163 邮箱 MCP：已完成部分授权测试，因 IMAP 网络延迟问题暂时暂停。

## 设计原则

个人副脑默认采用“本地优先、确认优先、可追踪优先”的设计。它会尽量自动完成低风险读取、分析和总结，但涉及写入、外部账号、浏览器会话、桌面控制、凭据和网络发布时，都必须先让用户知道并确认。

## 展示页面说明

`personal-brain-hud.html` 是个人副脑的可视化展示页和第一版 Jarvis 控制台。访客模式用于对外介绍能力和边界，本机模式用于显示本地模块状态和最近运行摘要。页面只读取 `brain-dashboard-data.json` 这样的只读快照，不直接读取数据库、钥匙串或真实邮件内容。

如果通过 `python3 -m brain_core jarvis serve` 启动本地服务，页面可以调用只读状态接口和规划接口；这些接口只生成计划，不直接执行工具。
