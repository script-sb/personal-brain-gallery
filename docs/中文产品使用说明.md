# 本地大脑产品使用说明

更新时间：2026-05-26

## 1. 这是什么

这是一个 GitHub-first 的本地智能体底座，目标是把 GitHub 上优秀的 AI/Agent 项目转化为你这台 Mac 上可控、可审计、可迭代的“本地大脑”。

它不是简单下载安装十个项目，而是做四件事：

- 管理 GitHub 候选项目和本地镜像；
- 保存个人/项目长期记忆；
- 提供本地代码和文件上下文搜索；
- 逐步接入浏览器调试、记忆引擎、代码语义搜索、虚拟工作区等能力。

当前默认安全策略是：读操作可以直接做，写文件、运行命令、浏览器会话、桌面操作、修改宿主配置都需要明确确认。

## 2. 当前可用能力

已经可用：

- 项目注册表：查看 16 个候选项目的状态、用途、风险。
- GitHub 镜像审计：查看哪些项目已 clone 到本机、对应 commit。
- 本地记忆：写入、搜索、导出、导入 SQLite 记忆。
- Agentmemory 适配器：已安装构建，但默认不接管记忆。
- Chrome DevTools MCP：已安装构建，可生成安全启动命令，并可 smoke-test。
- 本地上下文搜索：在目录内搜索文本内容。
- 端到端本地工作流：把记忆、上下文、工作区、计划和复盘串起来。
- 指数基金模块：生成配置模板、基金评分、再平衡建议。
- 任务计划记录：生成带确认策略的 run plan。

已镜像并构建/准备的核心模块：

| 模块 | 当前状态 | 用途 |
| --- | --- | --- |
| Chrome DevTools MCP | 已构建，可 smoke-test | 浏览器调试、console/network/DOM 诊断 |
| Agentmemory | 已构建，作为 opt-in adapter | 外部长期记忆引擎候选 |
| claude-context | 已构建，作为 opt-in adapter | 后续代码语义搜索 |
| Mirage | 已镜像，本地 facade 已可用 | 后续本地/虚拟工作区 |

## 3. 常用命令

所有命令都在项目根目录执行：

```bash
cd "/Users/apple/Documents/go up"
```

查看本地大脑状态：

```bash
python3 -m brain_core status
```

查看项目注册表：

```bash
python3 -m brain_core registry list
python3 -m brain_core registry matrix
```

查看 GitHub 镜像状态：

```bash
python3 -m brain_core github audit
```

生成镜像更新计划，不会直接执行：

```bash
python3 -m brain_core github mirror-plan --status integrate
```

## 4. 记忆功能

写入一条记忆：

```bash
python3 -m brain_core memory add "我希望所有高风险操作先确认" --tags preference,safety
```

搜索记忆：

```bash
python3 -m brain_core memory search 高风险
```

列出最近记忆：

```bash
python3 -m brain_core memory list
```

导出记忆到 JSON：

```bash
python3 -m brain_core memory export-json --output data/memories.backup.json
```

从 JSON 导入记忆：

```bash
python3 -m brain_core memory import-json data/memories.backup.json
```

替换式恢复记忆：

```bash
python3 -m brain_core memory import-json data/memories.backup.json --replace
```

查看 Agentmemory 适配器状态：

```bash
python3 -m brain_core memory adapter-status
```

生成 Agentmemory 服务命令：

```bash
python3 -m brain_core memory adapter-command --mode server
python3 -m brain_core memory adapter-command --mode mcp
python3 -m brain_core memory adapter-command --mode status
```

注意：当前 SQLite 是默认记忆源。Agentmemory 已构建，但不会自动启动，也不会自动运行 `connect codex` 等会修改宿主配置的命令。

## 5. 浏览器调试能力

查看 Chrome DevTools MCP 状态：

```bash
python3 -m brain_core devtools status
```

执行 smoke-test，确认 MCP 构建产物可启动：

```bash
python3 -m brain_core devtools smoke-test
```

生成默认安全启动命令：

```bash
python3 -m brain_core devtools command
```

连接已开启远程调试端口的 Chrome：

```bash
python3 -m brain_core devtools command --browser-url http://127.0.0.1:9222
```

默认会带上这些安全参数：

- `--no-usage-statistics`
- `--redact-network-headers`
- `--slim`

## 6. 本地上下文搜索

搜索当前项目：

```bash
python3 -m brain_core context search "Agentmemory" --path .
```

限制结果数量：

```bash
python3 -m brain_core context search "Chrome DevTools MCP" --path . --limit 5
```

这个能力目前是轻量文本搜索。后续会把 `claude-context` 升级为语义代码搜索适配器。

查看 claude-context 适配器状态：

```bash
python3 -m brain_core context adapter-status
```

执行 claude-context MCP smoke-test：

```bash
python3 -m brain_core context adapter-smoke-test
```

生成 claude-context MCP 命令：

```bash
python3 -m brain_core context adapter-command
```

如果未来要走本地 Ollama embedding + Milvus，可以生成类似命令：

```bash
python3 -m brain_core context adapter-command --provider Ollama --model nomic-embed-text --milvus-address localhost:19530
```

注意：当前只验证到 MCP 可启动。真正语义索引需要 embedding provider 和 Milvus，不会默认开启。

## 7. 任务计划记录

生成一个只读任务计划：

```bash
python3 -m brain_core task plan "分析当前本地大脑项目状态"
```

生成一个高风险任务计划：

```bash
python3 -m brain_core task plan "连接浏览器调试当前网页" --risky
```

高风险计划会标记为 confirm-required，提醒后续执行前需要确认。

查看 run 历史：

```bash
python3 -m brain_core runs list
```

查看某个 run 详情：

```bash
python3 -m brain_core runs show 2
```

查看某个 run 的产物：

```bash
python3 -m brain_core runs artifact 2
python3 -m brain_core runs artifact 2 --name run.json
```

## 8. 本地工作区

本地工作区是受 Mirage 启发的 local-only 原型。当前不启用任何远程连接器，只允许在项目内的 `workspace/` 目录读写。

查看状态：

```bash
python3 -m brain_core workspace status
```

执行 smoke-test：

```bash
python3 -m brain_core workspace smoke-test
```

写入文件：

```bash
python3 -m brain_core workspace write notes/hello.txt "hello local brain"
```

读取文件：

```bash
python3 -m brain_core workspace read notes/hello.txt
```

列出工作区：

```bash
python3 -m brain_core workspace list --path /
```

注意：这个能力当前只在 `workspace/` 内操作，防止误写系统目录。Mirage SDK 的真实接入留到后续依赖和安全审查之后。

## 9. 端到端本地工作流

本地工作流会把“记忆 -> 上下文搜索 -> 工作区产物 -> 计划 -> 复盘记忆”串起来。它不会自动执行浏览器、桌面、shell 或远程连接器动作，只生成可审计的产物。

运行一个本地任务：

```bash
python3 -m brain_core workflow local-task "总结当前本地大脑状态" --query "Local Brain"
```

标记为高风险任务：

```bash
python3 -m brain_core workflow local-task "准备调试一个网页问题" --query "DevTools" --risky
```

查看某次工作流：

```bash
python3 -m brain_core workflow show 1
```

每次运行会生成：

- `workspace/runs/<id>/summary.md`
- `workspace/runs/<id>/run.json`
- 一条 `workflow,run-log` 记忆

## 10. 指数基金模块

这个模块提供教育性配置建议，不替代持牌投顾的个性化投资建议。输出重点是：低成本、广泛分散、长期持有、定期再平衡。

根据期限和风险承受力生成画像：

```bash
python3 -m brain_core funds profile --years 10 --risk 3
```

生成组合模板：

```bash
python3 -m brain_core funds template --profile balanced
python3 -m brain_core funds template --profile growth --international-stock 30
```

给某只指数基金做基础评分：

```bash
python3 -m brain_core funds score --expense-ratio 0.03 --tracking-error 0.02 --assets-under-management 10000 --broad --liquid
```

查看是否需要再平衡：

```bash
python3 -m brain_core funds rebalance --profile balanced --stock 70 --bond 25 --cash 5
```

使用原则：

- 核心仓位优先考虑宽基指数，而不是窄主题。
- 优先比较费用率、跟踪误差、指数方法、流动性、税务效率。
- 长期目标可以承受更高股票比例，短期目标应降低波动。
- 交易前考虑税费、账户类型和现金流需求。

## 11. 当前风险与注意事项

Chrome DevTools MCP：

- npm audit 有 2 个 moderate 风险：`qs`、`ws`。
- 当前通过精简工具、关闭 usage statistics、redact network headers 降低风险。

Agentmemory：

- npm audit 有 3 high + 1 critical，集中在可选 embedding/transformers 链路。
- 当前只作为 opt-in adapter，不自动接管 SQLite 记忆。
- 不自动运行 `agentmemory connect ...`，避免修改 Codex/Claude/Cursor 等宿主配置。

claude-context：

- 已安装依赖并构建 MCP。
- `pnpm audit` 当前报告 90 个漏洞，其中 5 critical。
- 真实索引需要 embedding provider 和 Milvus，默认不自动索引任何私有代码。
- 当前仅作为 opt-in adapter，由用户确认路径和后端配置后再使用。

Mirage / Workspace：

- Mirage 已镜像，但还没有启用真实 SDK。
- 当前实现是 local-only facade，只写入 `workspace/`。
- 不启用 Gmail、Slack、S3、GitHub 等凭证型连接器。

第三方项目：

- `third_party/` 不进入 Git 版本管理。
- 不直接修改上游源码。
- 每个项目都应该先 mirror/audit，再 wrapper/integrate。

投资建议：

- 模块输出是教育性框架，不是保证收益或个性化买卖指令。
- 指数基金仍会亏损，股票指数在熊市可能出现大幅回撤。
- 任何实际买卖前，应结合收入、负债、税务、投资期限和紧急备用金。
