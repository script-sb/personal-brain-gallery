# Integration Roadmap

## Phase 1: Mirror And Audit

Done for the first `integrate` batch:

- `claude-context`
- `Agentmemory`
- `Chrome DevTools MCP`
- `Mirage`

## Phase 2: Wrapper Contracts

Create thin wrappers before installing or modifying upstream projects.

| Wrapper | Input | Output | Confirmation |
| --- | --- | --- | --- |
| DevTools | URL or active browser target | console/network/DOM summary | Browser session required |
| Memory | memory text, tags, query | stored/retrieved memory | Write confirmation for sensitive memories |
| Code Context | repo path, query | ranked source snippets | Read-only |
| Workspace | virtual mount spec, command | command output/artifact | Connector credentials required |

### Chrome DevTools MCP Status

Chrome DevTools MCP is the first wrapper candidate promoted from mirror to runnable local tool.

- Dependencies installed with `npm ci`.
- Build completed with `npm run build`.
- Local command is generated by `python3 -m brain_core devtools command`.
- Recommended defaults: `--no-usage-statistics --redact-network-headers --slim`.
- `npm audit` still needs a network-enabled rerun; initial install reported 2 moderate vulnerabilities.

### Agentmemory Status

Agentmemory is installed and built as an opt-in memory adapter.

- Dependencies installed with `npm install --legacy-peer-deps`.
- Build completed with `npm run build`.
- CLI help works via `node third_party/agentmemory/dist/cli.mjs --help`.
- Local command is generated by `python3 -m brain_core memory adapter-command`.
- SQLite remains the v1 source of truth until Agentmemory storage/privacy behavior is promoted.
- Do not run `agentmemory connect codex` or other host wiring commands without explicit confirmation; those commands mutate host configuration.

## Phase 3: Dependency Setup

Dependency install should be explicit and per project:

- Chrome DevTools MCP: inspect `npm install` footprint, then build/start MCP.
- Agentmemory: inspect local data location and privacy model before running.
- claude-context: inspect pnpm workspace and index output paths before indexing repos.
- Mirage: start with local RAM/Disk resources only; avoid Gmail/Slack/S3/GitHub credentials in v1.

## Phase 4: Local Brain Loop

The local brain should call wrappers in this order:

1. Retrieve local memories.
2. Retrieve project/context snippets.
3. Draft a plan.
4. Ask confirmation for risky tools.
5. Execute one tool step.
6. Record run log and durable memory.
